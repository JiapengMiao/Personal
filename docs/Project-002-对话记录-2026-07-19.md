## 2026-07-19 深夜：用户五项反馈修复

### 用户原话（2026-07-19 23:18，附 5 张看板截图）
> 1、国内的库存应该是上金所库存（1048.215吨）+上期所库存（990.039吨）=2038.254吨，你这里的计算不对吧？是不是用的不是010的数据？还是说数据是空值？交易所公布的数据应该是到7/10的，往后就没公布了，那后面几天的数据可以继续沿用最新数据，影响不大，等数据更新后再刷新就好了。
> 2、交易所库存趋势图这里，上金所的库存可以转换成日度的数据然后和上期所的做一张图，COMEX的库存太多了，不要和上期上金的放一块，然后还少了LBMA的库存，把LBMA的白银库存和COMEX的白银库存放一起做趋势图。
> 3、递延费方向这里有bug，然后下面的时间轴选取是摆设，拉动之后上面的时间区间不会变化的。
> 4、持仓量和虚实比没有用010的规则吗？只需要用偶数的合约对比就行了，白银只有偶数合约活跃，放奇数合约没意义。
> 5、基差和进口盈亏这里一样，下面的时间轴拉动之后不联动，虽然上面的图里数据是动了，但是下面的区间均值没有变化的

### 反馈与根因
1. **国内库存数字不对**：原实现只用单一交易所或含 null 简单相加；正确口径 = 上期所库存 + 上金所库存（用户给定 990.039 + 1048.215 = 2038.254 吨）。交易所数据只公布到各自截止日（上期所 7/17、上金所 7/10、LBMA 6/30），之后沿用前值
2. **交易所库存图混放**：COMEX（万吨级）与上期所/上金所（千吨级）同图导致国内线被压平；且缺 LBMA
3. **递延费图 bug**：tooltip 显示 undefined（旧 formatter 取错字段）；用户感觉"时间轴拉动不联动"实为竖线噪声图可读性差
4. **持仓量/虚实比合约冗余**：白银只有偶数月合约活跃，奇数合约（ag2607/2609/2611）无意义
5. **基差/盈亏统计不联动**：图上 dataZoom 能缩放，但"区间均值/百分位"统计卡写死全区间

### 改法
- **管道** `src/build_dashboard_data.py`：shfeInvT/sgeInvT/lbmaDailyT 前值填充（头部 null 保留）；domesticInvT = 两交易所 ffill 后相加，吨级序列 round 3；daily.json 新增顶层 `lastActual` 记录各序列真实截止日；positions_curve/virtual_ratio 过滤偶数月合约
- **前端**：Hero 国内库存徽章显示合计+构成小字（含"7/10 起沿用"动态标注）；市场区块 StocksPanel 重排国内/海外双小图；递延费改 ffill+连续 run markArea 色带（红=空付多、青=多付空，12% 透明度）+金色细阶梯线，tooltip 改 `params[0].name/value`；Basis.tsx 挂 `datazoom` 监听（batch.start/end → startValue/endValue 换算 → getOption 兜底三级读取），computeWindow 按窗口重算均值/百分位
