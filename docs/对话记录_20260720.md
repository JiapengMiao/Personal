## 五、核心教训

1. **ECharts 优化手段互相冲突**：LTTB × connectNulls:true = 假斜线+断裂；保真>流畅时原始数据直绘是兜底
2. **GitHub Pages 子路径部署**：vite 必须 `base: "./"`，fetch 必须用相对路径 `data/*.json`
3. **周频数据在日频轴上**：connectNulls:false 产生视觉虚线；单独 connectNulls:true（无 LTTB）是安全的
4. **数据维护在 Project-010**：管道 MAIN_XLSX 指向 Project-010 主表
5. **金银比单位**：USD/oz÷USD/oz 直接相除；CNY/g÷CNY/kg 要×1000
6. **RangePicker max**：必须取 max(数据末日期, 今天)
7. **git reset --hard 会丢未推送的修改**
8. **API 上传大 JS 文件可能导致内容损坏**：优先 git push，API 只做小文件补丁
