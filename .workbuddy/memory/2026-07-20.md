## 第二轮：修复拖动时间轴折线消失/散点化（用户截图反馈）
- 根因：`sampling: "lttb"` 与 dataZoom 拖动组合时，ECharts 对全量数据先降采样再缩放，缩放窗口内点被抽稀互不相连 → 视觉散点化；稀疏序列（国内库存早期）被采样把多年跨度的点连成斜线，出现"斜三角"假象。step:"end" 折线（递延费）被 LTTB 采样后会丢台阶。
- 修复：全项目删除 LTTB（MultiLineChart、DeferredChart、minuteOption、ProfitChart 共 4 处）；MultiLineChart 与 DeferredChart 新增默认缩放窗口——从"至少一条系列首个非空值"位置开始显示，消除 1968–2013 这类无数据死区。
- 05 持仓量与虚实比由 grid-2 改为 stack-grid 上下整行。
- 回归验证（src/verify_drag_20260720.py）：对 04 全部 4 图做三轮拖拽（中段/尾段小窗/拉回全量），canvas 绘制像素均保持 1.6 万+（修复前会掉到几百/零），无 page error。

## 第三轮：COMEX 图"虚线感"根因确认
- 用户再次反馈 COMEX 库存/注册仓单/PSLV 看起来像虚线。确认前端 lineStyle 本就无 dashed 配置，真正根因是 `connectNulls: false`：该面板三条线为周频/低频数据，落在日频 dates 轴上每周只有 1 个非空点、其余为 null；不跨空段连线时，每个真实值仅与相邻周的真实值相连，高密度下呈现"一段一段"的虚线观感。
- 修复：该面板改 `connectNulls: true`，并显式 `lineStyle.type: "solid"`。低频序列跨空段连线后成为连续实线。
- 方法论：ECharts 图"看似虚线"先别急着改 lineStyle.type，先检查 connectNulls 与数据频率是否匹配——周频数据放在日频轴上 + connectNulls:false = 视觉虚线。

## 第四轮：04 区块拖动 dataZoom 帧率优化
- 用户反馈：历史数据段拖动卡顿，最新日期附近流畅。根因：dataZoom 每帧对全量数据（1.4 万点）重布局 + 重绘；之前为修虚线去掉 LTTB 后失去减负手段。
- 修复（分层策略）：
  1. 数据量 > 4000 的图恢复 `sampling: "lttb"` —— ECharts 实际只对当前缩放窗口内数据采样，窗口越小点越稀，与 connectNulls:true 叠加不再产生虚线感；
  2. 全部 04 图 `animation: false` + `animationDurationUpdate: 0`，拖动时禁掉插值动画；
  3. dataZoom throttle 40ms → 80ms，拖动期间每秒最多 12 次完整重绘。
- 实测（src/verify_fps_20260720.py）：连续 15 次缩放操作 33–40 ops/秒，无 page error，视觉无降级。
- 经验：ECharts dataZoom 卡顿优化三件套 = 窗口内采样（sampling）+ 关动画（animation:false）+ 提高节流（throttle）；不要全局去采样，要在"保真"与"流畅"之间按数据量分层。

## 修复线上空白页（两层 404）
- 症状：https://jiapengmiao.github.io/Personal/ 打开空白。根因是**两层路径都以域根为基准**：
  1. vite 默认 `base: "/"`，构建出的 index.html 引用 `/assets/*.js`，被解析到 `jiapengmiao.github.io/assets/`（无 `/Personal` 前缀）→ JS 404 页面不启动。修复：vite.config 加 `base: "./"`。
  2. App.tsx / Basis.tsx 中 fetch 写死 `/data/*.json` 绝对路径 → 同样少了 `/Personal` 前缀 → 数据 404 页面停在 loading。修复：全部改为 `data/*.json` 相对路径（10 处）。
- 验证：无头浏览器实测线上页面 body 5528 字符、含"白银"、零 4xx 请求，截图确认完整渲染。
- **关键教训**：GitHub Pages 项目页（username.github.io/repo/）是子路径部署，前端所有资源引用（vite base + fetch 路径）都必须相对化；用 curl 看 200 不代表页面能跑，必须无头浏览器实测。只有部署在域根（自定义域名或 username.github.io 主仓库）才能用绝对路径。
